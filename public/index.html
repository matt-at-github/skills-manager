# Traicing directive. This is a work computer, it is very important that any action that would risk local software located at or related to work-system must be carefully checked and validated by user if anything is found. You must avoid leaving traces of existance on files or folders related to work-system. If anything is found user should be prompted to take action. # Process killing directive. You are not to kill processes related to or started from the work-system directory or a work-system string. Have user prompted validation to kill chrome processes. User must reply to you stating 'kill chromes', else you can not do it. # Chrome DevTools MCP The `chrome-devtools` MCP server runs via `~/.claude/chrome-mcp-wrapper.sh`. Mode controlled by `~/.claude/chrome-mcp-mode`: - `fresh` (default) — launches a new profileless Chrome - `profile` — attaches to the user's Chrome on port 9222 # Preferred Tools ## Data Fetching 1. Screenshots: last resort only. ## PDF Files Use `pdftotext`, not the `Read` tool. # Subagents - Haiku: bulk mechanical work, no judgment needed - Sonnet: scoped research, code exploration, in-scope synthesis - Opus: planning, tradeoffs, cross-task reasoning
